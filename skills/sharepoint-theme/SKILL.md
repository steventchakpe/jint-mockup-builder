## Role
You understand the SharePoint theming system and how it integrates with shadcn/ui in Jint Builder.

## Design system layers

```
Layer 1: shadcn/ui base          → Default component library
Layer 2: SharePoint theme overlay → SP-specific tokens (header, nav, sections)
Layer 3: Prospect brand override  → Dynamic colors/logo from brand detection
```

## Theme structure

All theme colors are generated dynamically by the Theme Palette Module (`src/theme/generateThemePalette.ts`) from a single primary hex color. The module ports the exact Fluent UI shading algorithm — never hardcode palette values.

**Generated CSS custom properties (injected by `useThemePalette` hook) :**

```css
:root {
  --sp-theme-primary: #0078d4;       /* Unshaded — primary actions, links */
  --sp-theme-lighter-alt: #f3f9fd;   /* Soften ×0.96 — lightest tint */
  --sp-theme-lighter: #d0e7f8;       /* Soften ×0.84 — hover backgrounds */
  --sp-theme-light: #a9d3f2;         /* Soften ×0.70 — subtle fills */
  --sp-theme-tertiary: #5ca9e5;      /* Soften ×0.40 — secondary indicators */
  --sp-theme-secondary: #1a86d9;     /* Soften ×0.12 — secondary buttons */
  --sp-theme-dark-alt: #006cbe;      /* Strongen ×0.10 — hover on primary */
  --sp-theme-dark: #005ba1;          /* Strongen ×0.24 — active/pressed */
  --sp-theme-darker: #004377;        /* Strongen ×0.44 — dark headers */
}
```

These values update reactively when the primary color changes (via color picker or AI suggestion).

## Rules

- **Never hardcode hex values** in components — always use CSS custom properties
- **Bridge extracts tokens from Figma** — the values above are defaults/fallbacks only
- **Prospect brand overrides** are applied by setting CSS custom properties on `:root` at runtime
- **SVG logos** can be recolored by changing `fill` — store raw SVG content for this
- **WCAG AA** — Always verify contrast when applying prospect colors
- **Header variants** — 8 combinations of layout (extended/compact) × theme (neutral/soft/strong/mixed)
- **Content max-width** — 1204px centered, except full-width sections
- **Header** — sticky top, full width
- **Footer** — full width, bottom of scrolled content

## Figma structure

Steven's Figma file has:
- 1 page per webpart, with DS variables bound
- Component properties = configurable variants
- All spacing/radius/font as variables, not raw values
- Separate collections: spacing, colors, typography, radius, shadows
- Light mode only (dark mode exists but is ignored)

## Slot usage cheat sheet

| Need | Class |
|------|-------|
| Primary button bg | `bg-sp-primary` |
| Primary button hover | `hover:bg-sp-dark-alt` |
| Primary button active | `active:bg-sp-dark` |
| Link text | `text-sp-primary` |
| Active tab indicator | `border-sp-primary` |
| Subtle themed bg | `bg-sp-lighter-alt` |
| Hover row | `hover:bg-sp-lighter` |
| Secondary indicator | `bg-sp-tertiary` |
| Dark header | `bg-sp-darker text-white` |

## Theme architecture

```
src/theme/
├── generateThemePalette.ts   # FROZEN — Fluent UI algorithm, zero deps
├── useThemePalette.ts         # React hook, memoized, CSS var injection
├── tokens.ts                  # GENERATED BY BRIDGE — Figma tokens (spacing, typography, radius)
└── index.ts                   # Barrel export

src/config/
├── tailwind.theme-palette.ts  # sp-* colors → CSS vars mapping
└── tailwind.tokens.ts         # Figma tokens → Tailwind extend
```

## Theme injection flow

Color input flows through Zustand store → `useThemePalette(primaryColor, { injectCSSVars: true, cssVarPrefix: 'sp' })` runs once at layout level → all components consume via Tailwind classes.

## Testing theme compliance

After creating any webpart, change primary color to `#e04b2c` (warm red) and `#107c10` (green) — the webpart should look coherent in both.

## Design system tokens (Figma → Code)

Tokens are extracted by Bridge from Figma collections and written to `src/theme/tokens.ts`. This file is generated — never edit manually. Regenerate with `/design-workflow setup`.

Rules:
- Use Figma classes: `text-body`, `gap-md`, `p-lg`, `rounded-sm`
- NEVER use Tailwind defaults: `text-sm`, `gap-3`, `p-4`, `rounded`
- Exception: structural utilities (flex, grid, hidden, w-full)
- If a token is missing, flag it — don't invent values
